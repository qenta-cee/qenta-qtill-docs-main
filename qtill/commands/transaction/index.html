Transaction Commands

Transaction Commands

INITIATEPAYMENT

Choose payment method and see the description for initiation of payment.

After every initiation, VERIFYPAYMENT needs to be done for asynchronous payment types.

VERIFYPAYMENT

This command must be used in conjunction with INITIATEPAYMENT for asynchronous payment types. The SUCCESSURL call itself says nothing about the (successful) outcome of the transaction.

Choose payment method and see the description for verifying payment.

Only by issuing this command the status of the transaction is checked on the system and in case of success the order/payment details are returned. In case of an error, more details are contained in the error parameters.
